# 本地智能选股系统

一个基于 Python 和 AKshare 数据接口的本地选股系统，用于自动筛选满足特定技术条件的 A 股股票。

## 功能特性

### 核心选股条件

系统基于以下三个条件进行股票筛选：

1. **周线金叉** 🔀
   - 周线 MA5 > MA10
   - 表示短期趋势向上

2. **涨停检测** 📈
   - 15个交易日内有过涨停
   - 涨幅阈值：≥9.5%（可配置）

3. **倍量交易** 📊
   - 5个交易日内有倍量交易
   - 成交量是前20日平均的2倍以上（可配置）

## 安装和使用

### 1. 环境要求

- Python 3.7+
- pip 包管理器
- 网络连接（用于获取数据）

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

主要依赖：
- akshare: 数据获取接口
- pandas: 数据处理
- numpy: 数值计算

### 3. 基础使用

#### 运行完整选股

检查所有 A 股并筛选符合条件的股票：

```bash
python main.py
```

#### 运行测试模式

仅检查前50个股票（用于快速测试）：

```bash
python main.py --sample 50
```

#### 检查指定股票

```bash
python main.py --codes 000001 000002 600000
```

### 4. 数据分析

对已选中的股票进行深度分析：

```bash
python analyze.py 000001
```

会输出：
- 趋势分析（最新价格、MA、52周高低点）
- 成交量分析
- 技术指标详情

### 5. 系统测试

测试各个模块的功能：

```bash
python test_system.py
```

## 配置说明

编辑 `config.py` 文件可调整选股参数：

```python
STOCK_FILTER_CONFIG = {
    'ma5_period': 5,              # MA5周期（默认5）
    'ma10_period': 10,            # MA10周期（默认10）
    'limit_up_days': 15,          # 检查涨停的天数（默认15）
    'volume_multiple_days': 5,    # 检查倍量的天数（默认5）
    'volume_multiple_ratio': 2.0, # 倍量比例（默认2.0倍）
    'volume_avg_period': 20,      # 平均成交量周期（默认20）
}

LIMIT_UP_THRESHOLD = 0.095       # 涨停判断阈值（默认9.5%）
```

### 指定检查的股票

如果不想检查所有 A 股，可以在 `config.py` 中设置：

```python
STOCK_LIST = ['000001', '000002', '000858']  # 仅检查这些股票
# STOCK_LIST = []  # 空列表表示检查所有A股
```

- **logs/stock_selector_YYYYMMDD.log** - 详细执行日志

## 工作原理

### 数据流程

1. **数据获取**
   - 使用 AKshare 接口获取所有 A 股列表
   - 逐个获取每个股票的日线和周线数据

2. **指标计算**
   - 计算各种技术指标（MA、MACD等）
   - 检查金叉条件
   - 检查涨停和倍量

3. **条件筛选**
   - 所有三个条件必须同时满足
   - 将符合条件的股票输出

4. **结果保存**
   - 将结果保存到 CSV 文件
   - 记录详细日志

### 技术指标详解

#### 周线金叉
- 基于周线数据计算 5 周期和 10 周期的简单移动平均
- 当 MA5 > MA10 时判断为金叉

#### 涨停检测
- 计算每日涨幅：$(收盘价 - 前日收盘价) / 前日收盘价$
- 在15个交易日内查找是否有涨幅 ≥ 9.5% 的日期
- 记录所有涨停日期和最大涨幅

#### 倍量检测
- 计算前20个交易日的平均成交量
- 在最近5个交易日内查找是否有日成交量 ≥ 平均值×2 的日期
- 记录所有倍量日期和最大倍数

## 常见问题

### Q: 运行时出现"网络错误"或"超时"

A: 
- 检查网络连接
- AKshare 数据接口可能暂时不可用
- 可以增加重试次数（在 config.py 中修改 MAX_RETRIES）

### Q: 为什么某个股票获取数据失败

A:
- 该股票可能是新上市股票，数据不足
- 股票代码可能不正确
- 该股票已退市或被特殊处理

### Q: 如何加速选股过程

A:
- 使用 `--sample` 参数进行测试
- 在 config.py 中指定具体要检查的股票列表
- 降低 REQUEST_TIMEOUT 配置

### Q: 能否修改选股条件

A:
- 可以在 config.py 中修改 STOCK_FILTER_CONFIG
- 也可以直接修改 stock_selector.py 中的条件逻辑
